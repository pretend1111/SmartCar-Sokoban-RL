# 第21届全国大学生智能汽车竞赛 · 智能视觉组

## 推箱子任务规则详解

> **版本说明**：本文档根据第21届全国大学生智能汽车竞赛智能视觉组（又称"人工智能视觉组"）官方发布的比赛细则及规则修改优化公告整理而成，涵盖推箱子任务的完整规则体系。

---

## 一、赛题概述

### 1.1 赛题背景

第21届智能车竞赛智能视觉组采用 **"虚实结合、硬件在环"** 的创新比赛模式。组委会基于 3D 游戏引擎开发了一套虚拟现实仿真系统，将经典的"推箱子"游戏与智能车竞赛相融合。

参赛车模在 **物理赛场** 上实际运动，系统通过全局摄像头实时捕捉车模位置和方向，并将其同步映射到 **虚拟游戏场景** 中。车模上搭载的屏幕会显示该虚拟场景的实时画面，参赛队伍需通过控制车模完成虚拟场景中的推箱子任务。

### 1.2 赛题目标

> **核心目标**：按照规则将所有箱子推到指定目的地，以最短时间完成任务。

本赛题是"推箱子"游戏的演变版本，比传统推箱子游戏有更多要求：
- 车模需要 **识别** 箱子上的图案/数字内容
- 根据识别结果 **决策** 将箱子推到正确的目的地
- 游戏难度和要求随 **关卡递增**
- 场景中可能包含 **炸弹** 等障碍物需要避开

### 1.3 组别定位

智能视觉组面向未来智能驾驶系统的关键能力培养，鼓励应用以下先进技术：
- 多模态视觉识别
- 强化学习
- 智能体规划与决策

## 二、比赛场地

### 2.1 场地尺寸与布局

| 参数 | 规格 |
|------|------|
| 场地尺寸 | 约 **3.2m × 2.4m**（与 320×240 图像分辨率比例一致） |
| 边界标识 | 宽度 **5cm** 的黄色布基胶带 |
| 可活动区域 | 黄色胶带内部 3.2m × 2.4m |

### 2.3 虚拟围墙系统

虚拟场景中包含 **围墙** 元素，是推箱子地图的重要组成部分：

- 围墙在虚拟场景中显示为 **黑灰色**
- 围墙分为 **外围围墙** 和 **中间围墙** 两类
- **外围围墙**：场地最外圈的围墙，**不可被炸毁**
- **中间围墙**：场地内部的围墙，可通过炸弹元素 **炸毁**，从而缩短运动路线
- 如果车模的真实坐标穿过围墙，虚拟图像会停留在未穿过的位置，导致车模 **无法获取正确坐标**
- 车模需从上次离开围墙的附近回到游戏中，系统才会重新识别并刷新位置
- ⚠️ **如果车模直接穿过最外围围墙，比赛自动结束**

### 2.4 场地定位系统

场地四个角落贴有 **AprilTag 定位码**（编号 15、16、17、18），用于：
- 系统自动定位场地位置
- 进行全局逆透视校正
- 定位码高度统一为 **15cm**（与车模标识牌同高）

### 2.5 全局摄像头

场地正中央高处安装一个 **全局快门彩色摄像头**，具有以下特性：
- 高帧率采集
- 无畸变镜头
- 用于实时识别场地内所有车模的位置和朝向

### 2.6 大屏显示系统

| 参数 | 规格 |
|------|------|
| 屏幕尺寸 | 约 **75英寸** |
| 用途 | 实时播放虚拟场景图像，供观众和选手查看 |
| 每视角分辨率 | **720 × 540** |
| 渲染效果 | 车模以车辆形象呈现，增加音效、特效和背景音乐 |

## 三、车模规格要求

### 3.1 车模型号

允许使用以下成品车模型号：**B、C、F、G、M、H、Y、Z** 等。

> 推荐使用 **Y型车模**（新一代 AI 模型组指定车模）。

### 3.2 尺寸限制

| 参数 | 规格 |
|------|------|
| 地面投影 | 车模及附属物投影需在 **35cm × 35cm** 范围内 |
| 车模高度 | 无具体限制 |
| 推箱机构 | 用于推动箱子的机械结构必须在底盘圆形边界内 |
| 摄像头支架 | 不受底盘尺寸限制 |

### 3.3 方位标识牌

方位标识牌是车模被系统识别的关键部件：

| 参数 | 规格 |
|------|------|
| 尺寸 | **30cm × 15cm** 长方形 |
| 组成 | 红色矩形 + AprilTag码 + 灰绿色矩形 |
| AprilTag类型 | TAG16H5 系列，编号 01-05（默认使用编号 1） |
| 灰绿色颜色 | RGB(90, 179, 89) / CMYK(65, 3, 81, 0) |
| 安装高度 | 固定 **15cm**（±0.5cm） |
| 安装方式 | **磁吸设计** |
| 铁片间距 | 两铁片中心间距 **20cm**（±0.2cm） |
| 安装要求 | 水平放置，上方不能有折叠，与车模之间无相对位移或转动 |

> **注意**：参赛选手需根据所选车模 **自行制作支架并预装磁铁**，比赛现场将提供官方标识牌供磁吸安装。

### 3.4 车载显示设备

车模上需搭载以下任一显示终端：
- **普通手机**
- **无线投屏器/无线屏幕**

显示内容由裁判系统无线投屏，包括：
- 比赛场地的 **全局图像**
- 车模的 **第一视角图像**

### 3.5 电子元器件限制

| 类别 | 要求 |
|------|------|
| 主控芯片 | 限定 **NXP 公司 MCU**（推荐 i.MX RT 系列） |
| 图像处理平台 | 限定使用 OpenART / OpenART Plus / MCXVision / MV5-RT / OMV-RT 模块（均基于 NXP MCU） |
| 普通传感器 | **无限制** |

## 四、推箱子任务详细规则

### 4.1 虚实结合机制

```
┌─────────────────────────────────────────────────────────────┐
│                     比赛系统架构                              │
│                                                             │
│  ┌──────────┐    全局摄像头     ┌──────────────┐             │
│  │ 物理场地  │ ──────────────→ │  裁判系统     │             │
│  │ (车模运动) │                 │ (位置识别)    │             │
│  └──────────┘                  └──────┬───────┘             │
│                                       │                     │
│                                       ▼                     │
│                               ┌──────────────┐             │
│                               │ 虚拟游戏引擎  │             │
│                               │ (推箱子场景)  │             │
│                               └──────┬───────┘             │
│                                       │                     │
│                          ┌────────────┼────────────┐       │
│                          ▼            ▼            ▼       │
│                    ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│                    │ 车载屏幕  │ │  大屏显示 │ │ 录制系统  │  │
│                    │ (投屏)   │ │ (75寸)   │ │          │  │
│                    └──────────┘ └──────────┘ └──────────┘  │
└─────────────────────────────────────────────────────────────┘
```

**工作流程**：
1. 全局摄像头识别车模物理位置和朝向（通过 AprilTag 标识牌）
2. 裁判系统将车模位置映射到虚拟场景中（如红色箭头 = 1号车模，绿色箭头 = 2号车模）
3. 虚拟游戏引擎渲染推箱子场景
4. 实时画面通过无线投屏到车载显示设备
5. 车模上的摄像头观察投屏画面，获取游戏状态信息
6. 车模根据识别结果做出决策并在物理场地上运动
7. 运动再次被映射回虚拟场景，形成闭环

### 4.2 箱子类型

场地中存在两类箱子：

#### 4.2.1 可移动箱子（待推箱子）

- 外形：**正方体**
- 顶部：保留原有颜色
- 四周侧面：贴有需要 **识别的图案/内容**
- 四面图案可能不完全相同，但 **类型一致**
- 由裁判 **随机放置** 在场地中，无固定方向
- 箱子之间保持至少 **50cm** 的间距

#### 4.2.2 目的地箱子（目标位置）

- 四周侧面：贴有 **数字**
- 作为车模推箱子的目标区域/标记

### 4.3 识别与分类任务

车模需要识别箱子上的内容，并根据识别结果决定推送方向：

#### 场景一：物品分类

箱子上的图案分为两大类：
| 类别 | 推送方向 |
|------|----------|
| **电子外设**（如电路板、芯片、传感器等） | 推向赛道指定一侧 |
| **常用工具**（如螺丝刀、扳手、万用表等） | 推向赛道另一侧 |

#### 场景二：数字识别

当箱子上的内容为 **手写数字**（0-99 范围）时：
| 数字特征 | 推送方向 |
|----------|----------|
| **奇数** | 推向赛道 **左侧** |
| **偶数** | 推向赛道 **右侧** |

#### 识别结果记录

- 车模必须 **按顺序记录** 每个箱子的识别结果（小类名称或数字）
- 比赛结束后，识别序列需在 **车自带屏幕上显示**
- 供裁判 **核对验证**

### 4.4 推箱动作要求

1. 车模沿赛道/场地行驶，**搜索并发现** 箱子
2. 接近箱子后进行 **图像识别**
3. 根据识别结果 **判断推送方向**（左/右）
4. 将箱子 **推离赛道**（推出场地边界或推至目的地）
5. 继续搜索下一个箱子，直至所有箱子处理完毕

> **重要**：用于推动箱子的机械结构必须保持在底盘圆形边界范围内。

### 4.5 推箱子胜利条件

当可移动箱子被推到对应的目的地后，**箱子和目的地会同时消失**，表示该箱子任务完成。所有箱子消失即为通关。

### 4.6 关卡与难度递进

竞赛计划了 **三种难度等级** 的游戏供选择：

| 难度 | 特征 |
|------|------|
| 初级 | 纯推箱子，无额外识别要求 |
| 中级 | 需根据箱子图案分类推送 |
| 高级 | 分类推送 + 更复杂的地图布局 |

后续关卡可能在以下方面增加难度：
- 箱子数量增多
- 地图中出现更多 **围墙**，路线更复杂
- 场地中出现 **炸弹元素**（可炸毁中间围墙缩短路线，但触碰可能有风险）
- 识别内容更加复杂
- 推送规则更加多样
- 箱子可能被推入 **角落死角**，导致无法继续推动

## 五、比赛流程

### 5.1 赛前准备

1. 参赛队伍将车模放置于物理场地的 **指定待命区/起跑线后方**
2. 确保车模标识牌正确安装（磁吸固定，高度 15cm）
3. 车载显示设备就绪，可正常接收投屏画面
4. 在裁判系统中设置 **自定义玩家名称**
5. 根据需要调节摄像头参数

### 5.2 比赛开始

1. 裁判确认场地箱子摆放完毕（由 4 名辅助裁判负责立方体摆放）
2. 主裁判下达发车指令
3. 车模从起跑线后出发，**系统自动开始计时**
4. 虚拟场景同步启动

### 5.3 比赛进行中

- 每支队伍有 **15-25 分钟** 的比赛时间
- 允许进行 **现场程序调试**
- **禁止人工干预** 车模运行
- 需要 **5 名裁判** 协作（1 名主裁判 + 4 名辅助裁判）

### 5.4 比赛结束条件

比赛在以下任一条件满足时结束（计时停止）：

| 条件 | 说明 |
|------|------|
| ✅ 任务完成 | 所有箱子成功推至目的地并消失 |
| ⚠️ 箱子死锁 | 场上剩余箱子已被推入角落，无法继续推动 |
| ❌ 穿越外围围墙 | 车模穿过最外围围墙，**比赛自动结束** |
| ❌ 超时 | 超过比赛时间上限 |
| ❌ 意外停止计时 | 车模动作过大意外触发计时器停止，比赛直接结束 |

## 六、评分与计时规则

### 6.1 基本评判原则

> **以完成整体任务所需的时间长短来判定胜负，用时最短者获胜。**

### 6.2 罚时规则

| 违规情况 | 罚时 |
|----------|------|
| 将箱子推错方向 | **+30 秒** / 次 |
| 未能将箱子完全推离赛道 | **+30 秒** / 次 |
| 识别错误（记录结果与实际不符） | **+30 秒** / 次 |
| 搜索遗漏（未处理的箱子） | **+30 秒** / 次 |

### 6.3 特殊情况

- 未完成 **60%以上** 的识别和搬运任务直接返回，将按罚时标准处罚
- 车模未返回车库或两轮及以上冲出车库，视为入库失败，**+15 秒**

### 6.4 最终成绩计算

```
最终成绩 = 实际完成时间 + 罚时总计
```

## 八、规则修改要点（第21届新增/变更）

针对第21届竞赛，智能视觉组规则进行了以下优化修改：

### 8.1 车模标识牌统一

| 修改前 | 修改后 |
|--------|--------|
| 标识牌安装方式各异 | 统一采用 **磁吸设计** |
| 安装高度 ≤ 15cm | 固定高度 **15cm** |
| 标识牌颜色为黄色 | 更改为 **灰绿色** RGB(90, 179, 89) |

> **修改原因**：黄色标识牌与场地黄色边框冲突，影响系统识别。

### 8.2 AprilTag 定位码高度调整

| 修改前 | 修改后 |
|--------|--------|
| 定位码高度不统一 | 统一调整为 **15cm** |

> **修改原因**：使定位码与车模标识牌同高，解决虚拟车模定位偏差问题。

### 8.3 箱子图案规则优化

- 可移动箱子侧面贴有识别图案（四面可不同但类型一致）
- 目的地箱子侧面贴有数字
- 箱子顶部保留原有颜色，仅侧面有图案

## 九、技术路线建议

> 以下为参考性技术方案建议，非官方规则内容。

### 9.1 视觉感知

```
车载摄像头 → 捕获投屏画面 → 图像预处理 → 目标检测（箱子定位）
                                           → 图像分类（物品/数字识别）
```

### 9.2 关键技术点

| 技术环节 | 建议方案 |
|----------|----------|
| 箱子检测 | 颜色分割 + 轮廓检测，或基于深度学习的目标检测 |
| 图案识别 | CNN 图像分类（物品类别），手写数字识别模型 |
| 路径规划 | A* / BFS 等传统搜索算法，或强化学习 |
| 推箱策略 | 有限状态机 / 行为树，避免死角和死锁 |
| 定位导航 | 利用虚拟场景中的全局/第一人称视角信息 |
