# **LAMP-Net：面向 MCU 实时推箱子的轻量级神经—符号决策架构**

LAMP = **Legal-action masked, Assignment-aware, Macro-action Policy Network**。核心思想是：**不要让神经网络从零学习推箱子规则、BFS、死角、ID 排除这些硬逻辑；网络只学习“在合法候选动作里哪个更像专家的长期决策”。** 这样最符合这份 RFC 的硬约束：16×12 网格、部分可观测 ID、炸弹/内墙、OpenART mini 无 GPU/NPU、INT8 TFLite Micro 部署、决策模型 ≤500 KB、≤50 ms。

---

## 0. 总体结论

我不建议做端到端大模型、Transformer、RNN 或纯 CNN 行为克隆。最优方案应是：

```text
[状态维护 / ID belief / 规则引擎]
        ↓
[≤1 ms 经典特征：BFS、可达性、推送距离场、死角、FOV 信息增益]
        ↓
[LAMP-Net：轻量 CNN + 全局 MLP，输出 push/view/value]
        ↓
[合法动作 mask + 安全过滤 + 一步符号模拟]
        ↓
[BFS 路径规划器生成低层动作]
        ↓
[实车闭环控制]
```

这个系统不是“神经网络替代规划”，而是**神经网络给规划排序**。这和 Neural A*、Value Iteration Networks 这类把神经网络用于规划引导的思想一致：神经网络学启发式/策略，经典搜索保证动作合法性与结构化推理。Neural A* 的核心就是学习 guidance map 来提升 A* 搜索效率与路径质量；VIN 则把近似 value iteration 表达成 CNN 形式，用于规划型任务。([arXiv][1])

# 1. 为什么当前 baseline 不够

RFC 里的 baseline 是 hybrid CNN + MLP，输出固定离散动作，再用浅层 branch search 修正。它在 Phase 4 之后明显掉速。根因不是模型不够大，而是**归纳偏置不对**：

1. **固定 54 类动作头太粗糙**
   推箱子真正动作是“某个格子的某个实体，沿某个方向推”。动作空间天然是空间化的：
   [
   a = (x, y, d)
   ]
   用固定类别会破坏平移对称性，也难以泛化到不同箱子位置。

2. **把规则交给网络学是浪费容量**
   墙、可达性、推送合法性、死角、BFS 距离都可以精确算出。让小 CNN 自己从数据里学这些，会消耗大量样本和参数。

3. **部分可观测 ID 没有被显式建模**
   未识别箱子/目标的 ID 不是普通缺失值，而是一个带排除推理的 belief state。若不显式维护，模型会把“未知 ID”误当作固定类别，导致探索策略不稳定。

4. **纯 BC 有分布偏移**
   模型一旦走到专家轨迹外的状态，BC 没见过这些状态，错误会累积。DAgger 正是为解决这种 imitation learning 的 covariate shift：让当前策略跑出自己的状态分布，再查询专家并加入训练集。([arXiv][2])

# 2. 状态编码

地图裁掉外圈墙，只保留真正可玩区域：

[
H = 10,\quad W = 14,\quad N = 140
]

输入不是原始图像，而是**游戏状态张量 + 经典规划特征**。YOLO 只负责识别 ID，决策模型不直接看相机图像。

## 2.1 Grid tensor

定义：

[
X \in \mathbb{R}^{10 \times 14 \times C}
]

建议 (C = 48)。每个通道含义如下。

### A. 基础地图通道，8 个

| 通道               | 含义             |
| ---------------- | -------------- |
| wall_static      | 永久墙 / 内墙       |
| wall_destroyable | 当前仍存在的可炸内墙     |
| empty            | 可通行空地          |
| player           | 车当前位置 one-hot  |
| player_reach     | 当前车可达区域        |
| fov_visible      | 当前 FOV 可见区域    |
| dead_corner      | 箱子进入后不可出的死角    |
| garage/start     | 起点/车库位置，如果需要入库 |

### B. 动态实体通道，10 个

| 通道                  | 含义             |
| ------------------- | -------------- |
| box_present         | 箱子位置           |
| target_present      | 目标位置           |
| bomb_present        | 炸弹位置           |
| object_blocking     | 会挡路的实体：箱子/炸弹   |
| box_known_id        | 该格箱子 ID 是否已知   |
| target_known_id     | 该格目标 ID 是否已知   |
| box_unknown         | 未知 ID 箱子       |
| target_unknown      | 未知 ID 目标       |
| bomb_explodable     | 炸弹是否可被推爆       |
| pair_completed_mask | 已完成/已消除辅助历史，可选 |

### C. ID belief 通道，20 个

箱子 ID 与目标 ID 各 10 维 one-hot/belief：

[
P_b(i, k) = \Pr(\text{box}_i\text{ has ID }k)
]

[
P_t(j, k) = \Pr(\text{target}_j\text{ has ID }k)
]

若已识别，则为 one-hot；若未知，则按剩余 ID 集合均匀分布；若可由排除法推出，也直接变 one-hot。

渲染到 grid 上：

| 通道                          | 含义                     |
| --------------------------- | ---------------------- |
| box_id_0 ... box_id_9       | 若该格有箱子，填入该箱子 ID belief |
| target_id_0 ... target_id_9 | 若该格有目标，填入目标 ID belief  |

这样网络不需要自己“猜”排除法，输入里已经是 belief state。

### D. 规划特征通道，10 个

这些由经典算法 ≤1 ms 算出：

| 通道               | 含义                  |
| ---------------- | ------------------- |
| player_bfs_dist  | 车到每格最短步数，归一化到 [0,1] |
| push_dist_min    | 任意箱子到匹配目标的最短推送距离    |
| push_dist_grad_u | 向上推后距离是否变小          |
| push_dist_grad_d | 向下推后距离是否变小          |
| push_dist_grad_l | 向左推后距离是否变小          |
| push_dist_grad_r | 向右推后距离是否变小          |
| legal_push_u     | 当前是否能从该格向上推         |
| legal_push_d     | 当前是否能从该格向下推         |
| legal_push_l     | 当前是否能从该格向左推         |
| legal_push_r     | 当前是否能从该格向右推         |

这里的关键是：**网络不需要学“这个动作合法吗”，它只需要学“合法动作里哪个更好”。**

# 3. ID 配对与 belief 机制

部分可观测性是本题难点之一。我的建议是：**ID 推理不交给神经网络，显式维护 belief matrix。**

设当前有 (m) 个箱子、(m) 个目标。

箱子 belief：

[
B \in [0,1]^{m \times 10}
]

目标 belief：

[
T \in [0,1]^{m \times 10}
]

箱子 (i) 与目标 (j) 匹配概率：

[
M_{ij} = \sum_{k=0}^{9} B_{ik}T_{jk}
]

若 ID 完全已知，(M) 就退化成 0/1 匹配矩阵。若部分未知，(M) 提供软配对信息。

进一步定义箱子 (i) 的期望目标距离：

[
D_i = \sum_{j=1}^{m} M_{ij} \cdot d_{\text{push}}(box_i, target_j)
]

其中 (d_{\text{push}}) 由反向 BFS 得到。这个值直接作为 grid 特征渲染到箱子所在格。

这样做的好处是把 (N!) 的配对爆炸压缩成 (N^2) 的软匹配矩阵。最大箱子数 5 时，复杂度极低。

# 4. 动作空间设计

我建议不用 54 类固定动作，而是空间化 macro-action。

## 4.1 推箱/推炸弹动作

正交推送：

[
a_{\text{push}} = (x, y, d),\quad d \in {U,D,L,R}
]

输出张量：

[
Z_{\text{push}} \in \mathbb{R}^{10 \times 14 \times 4}
]

其中 (Z[x,y,d]) 表示“推动位于 ((x,y)) 的箱子/炸弹，方向为 (d)”的 logit。

炸弹对角爆破可以作为额外动作：

[
a_{\text{bombdiag}} = (x, y, d_{\diag})
]

但我更建议不要单独开 head，而是在规则引擎中把对角爆破转化成一个合法候选动作，然后复用 push score，加一个 bomb bonus 特征。

## 4.2 探索/观察动作

当 ID 未知时，系统不应该盲目推箱，而应该先到某个观察位姿识别 ID。

定义观察动作：

[
a_{\text{view}} = (x, y, o),\quad o \in {0,1,\ldots,7}
]

输出张量：

[
Z_{\text{view}} \in \mathbb{R}^{10 \times 14 \times 8}
]

表示“车移动到 ((x,y))，朝向 (o)，用于观察未知实体”的价值。

规则引擎枚举所有能看到未知箱子/目标的候选位姿，并计算信息增益：

[
IG(a) = H(\text{belief before}) - H(\text{belief after expected observation})
]

最终探索动作打分：

[
S_{\text{view}}(a) =
Z_{\text{view}}(a)

* \alpha IG(a)

- \beta d_{\text{walk}}(player, a)
  ]

这样可以显式避免“模型为了通关而忽略识别”的问题。

## 4.3 动作解码

运行时：

```text
1. 规则引擎枚举所有 legal push/view actions
2. 对非法动作 mask 为 -∞
3. 对危险动作做 hard filter：
   - 非目标格死角推入：禁止
   - 明显推远且无收益：降权
   - 未知 ID 箱子推向确定目标：禁止或强惩罚
4. 选择得分最高的 macro-action
5. BFS 把 macro-action 翻译为低层移动
```

最终得分：

[
S(a) =
Z_\theta(a)

* \lambda_1 d_{\text{walk}}(a)
* \lambda_2 R_{\text{deadlock}}(a)
* \lambda_3 R_{\text{unknown}}(a)

- \lambda_4 \Delta h_{\text{classical}}(a)
  ]

其中 (\Delta h_{\text{classical}}) 是推完后经典启发式是否变好，例如剩余推送距离是否下降。

# 5. 网络结构

模型必须适配 TFLite Micro 与 CMSIS-NN。CMSIS-NN 本身就是为 Arm Cortex-M 提供高效神经网络 kernel 的库，支持 int8/int16 量化规范并针对 Cortex-M SIMD 等能力优化；因此结构上应尽量使用 Conv2D、DepthwiseConv2D、FullyConnected、ReLU、Add、Pooling 这些高兼容算子。([arm-software.github.io][3])

## 5.1 LAMP-Net 结构图

输入：

[
X: 10 \times 14 \times 48
]

网络：

```text
Input 10×14×48
  ↓
1×1 Conv, 48 → 48, ReLU
  ↓
[Depthwise 3×3 Conv + 1×1 Pointwise Conv + ReLU + residual] × 8
  ↓
Feature map F: 10×14×48
  ├── Spatial push head: 1×1 Conv 48→4
  ├── Spatial view head: 1×1 Conv 48→8
  └── Global average pooling
          ↓
      concat global scalar features g
          ↓
      FC 112→128, ReLU
          ↓
      FC 128→128, ReLU
          ├── FC 128→560 reshape to 10×14×4, add to push head
          ├── FC 128→1120 reshape to 10×14×8, add to view head
          ├── FC 128→4 mode head
          └── FC 128→1 value head
```

为什么 push/view head 同时有 spatial conv 和 global FC？

因为推箱子动作既依赖局部几何，也依赖全局任务阶段。
例如“这个箱子旁边能推”是局部信息；“现在应该先炸墙还是先识别 ID”是全局信息。

所以最终：

[
Z_{\text{push}} =
Z_{\text{push}}^{\text{local}}
+
Z_{\text{push}}^{\text{global}}
]

[
Z_{\text{view}} =
Z_{\text{view}}^{\text{local}}
+
Z_{\text{view}}^{\text{global}}
]

## 5.2 全局标量特征

额外输入：

[
g \in \mathbb{R}^{64}
]

包含：

| 特征                                     | 维度  |
| -------------------------------------- | --- |
| 剩余箱子数 one-hot                          | 5   |
| 剩余炸弹数 one-hot                          | 4   |
| 已识别箱子数                                 | 1   |
| 已识别目标数                                 | 1   |
| belief entropy                         | 1   |
| 当前 phase one-hot，训练时用，部署可不用            | 6   |
| 当前总步数 / 剩余步数                           | 2   |
| 全局最小推送距离统计                             | 4   |
| 可达区域面积                                 | 1   |
| legal push 数量                          | 1   |
| deadlock risk 统计                       | 4   |
| ID matching matrix 上三角/flatten padding | ≤25 |
| 其他 padding                             | 若干  |

部署时可固定 64 维。

## 5.3 参数量估计

粗略估算：

| 模块                             |            参数量 |
| ------------------------------ | -------------: |
| Stem 1×1 Conv 48→48            |          2.3 K |
| 8 个 depthwise-separable blocks |         21.9 K |
| Spatial heads                  |          0.6 K |
| Global FC 112→128→128          |         30.7 K |
| Push global head 128→560       |         71.7 K |
| View global head 128→1120      |        143.4 K |
| Mode/value/aux heads           |            2 K |
| **总计**                         | **约 273 K 参数** |

大小：

| 格式                            |           大小 |
| ----------------------------- | -----------: |
| fp32                          |     约 1.1 MB |
| int8                          |     约 273 KB |
| 加 TFLite metadata / alignment | 约 300–360 KB |

满足 500 KB 限制。

激活 RAM 也很低。最大 feature map：

[
10 \times 14 \times 48 = 6720
]

INT8 激活约 6.7 KB。即使 TFLite Micro arena 有中间 buffer、im2col、alignment，预计仍远低于 2 MB。

# 6. 训练范式

## 6.1 不只做 BC，而做三阶段训练

### Stage A：专家行为克隆预训练

数据来源：

1. IDA*：质量最高，用于 Phase 1–4 及部分 Phase 5/6 难图。
2. BestFirst：覆盖 Phase 5/6 大量样本。
3. AutoPlayer：只用于探索行为和低难度 warm-up，不作为最终最优策略主标签。

损失：

[
\mathcal{L}_{BC}
================

-\sum_{a \in A_{\text{legal}}} q(a)\log \pi_\theta(a|s)
]

其中 (q(a)) 不是单一 one-hot，而是 soft expert target：

[
q(a)
====

\frac{\exp(-C^*(s,a)/\tau)}
{\sum_{a'}\exp(-C^*(s,a')/\tau)}
]

如果某状态下多个动作都接近最优，模型不应该被强迫只学一个动作。

### Stage B：Value/Q 辅助学习

训练 value head：

[
V_\theta(s) \approx -C^*(s)
]

其中 (C^*(s)) 是专家剩余总代价，包含推数和车走步数。

Huber loss：

[
\mathcal{L}_{V}
===============

\text{Huber}(V_\theta(s), -C^*(s))
]

同时训练 action advantage：

[
A^*(s,a)
========

C^*(s) - C^*(T(s,a))
]

这样模型不只是模仿动作，而是学习“这个动作会不会让局面变好”。

### Stage C：DAgger / on-policy 修正

纯 BC 最大问题是 rollout 后偏离专家分布。DAgger 的做法是：让当前策略自己跑，遇到状态后再询问专家，把这些状态加入训练集。这个思想非常适合本题，因为仿真器已有，BestFirst 也能在几秒到几十秒内给标签。([arXiv][2])

流程：

```text
1. 用专家轨迹训练初版 LAMP-Net
2. 在全部 phase 上跑 deterministic rollout
3. 收集失败前 5–20 步状态、低置信度状态、死锁前状态
4. 用 BestFirst 标注；特别难的状态夜间用 IDA*
5. 加入 replay buffer
6. 重训 / fine-tune
7. 重复 3–5 轮
```

这一步对 Phase 4–6 最关键。

# 7. 总损失函数

最终损失：

[
\mathcal{L}
===========

\mathcal{L}*{push}
+
\lambda_1 \mathcal{L}*{view}
+
\lambda_2 \mathcal{L}*{mode}
+
\lambda_3 \mathcal{L}*{V}
+
\lambda_4 \mathcal{L}*{legal}
+
\lambda_5 \mathcal{L}*{deadlock}
+
\lambda_6 \mathcal{L}_{distill}
]

建议权重：

| loss            |  权重 |
| --------------- | --: |
| push CE         | 1.0 |
| view CE         | 0.7 |
| mode CE         | 0.5 |
| value Huber     | 0.2 |
| legal BCE       | 0.1 |
| deadlock BCE    | 0.2 |
| distillation KL | 0.3 |

其中：

* (\mathcal{L}_{legal})：辅助预测 legal push mask。
* (\mathcal{L}_{deadlock})：预测某动作是否导致死锁。
* (\mathcal{L}_{distill})：从大 teacher 模型蒸馏到小模型。

Teacher 可以是训练机上的大模型，例如 ResNet-like CNN + GNN + search，不部署，只负责生成软标签。

# 8. 数据增强

## 8.1 几何对称增强

由于地图是 14×10，不建议做 90° 旋转，因为会变成 10×14。推荐：

1. 水平翻转
2. 垂直翻转
3. 180° 旋转

同时变换方向标签：

[
U \leftrightarrow D,\quad L \leftrightarrow R
]

车朝向、FOV、推送方向、view orientation 都要同步变换。

## 8.2 ID permutation 增强

随机重命名 ID：

[
k \mapsto \sigma(k)
]

只要箱子和目标一起改，任务本质不变。这能防止模型记住“数字 3 比数字 7 更特殊”这种伪规律。

## 8.3 部分可观测增强

训练时随机 mask 一部分已知 ID，模拟识别延迟：

```text
p = 0.1–0.3 随机把已知 ID 改成 unknown belief
```

这样模型会学会更稳健的探索策略。

## 8.4 反事实负例

对每个专家状态，采样若干非专家合法动作：

* 推入死角
* 把箱子推离目标
* 未识别时乱推
* 炸弹推错墙
* 让路径被箱子堵死

把这些动作标为低分，训练 action-ranking loss：

[
\mathcal{L}_{rank}
==================

\max(0, m + S(a^{-}) - S(a^{+}))
]

# 9. 课程策略

推荐训练顺序：

| 阶段           | 数据                 | 目标                 |
| ------------ | ------------------ | ------------------ |
| Curriculum 0 | 人工生成极简 1 箱无墙       | 学基本 push-to-target |
| Curriculum 1 | Phase 1–2          | 学 BFS 特征、目标靠近      |
| Curriculum 2 | Phase 3–4          | 学多箱顺序、避免互相阻挡       |
| Curriculum 3 | Phase 5            | 学炸弹、内墙变化           |
| Curriculum 4 | Phase 6            | 混合困难局              |
| Curriculum 5 | DAgger hard states | 修正失败分布             |

采样上不要按数据量平均，而要 hard-example weighted：

[
p_i \propto 1 + \gamma \cdot \mathbb{1}[\text{policy failed on } i]
]

Phase 6 和失败样本必须过采样，否则模型会被 Phase 1–3 的简单样本淹没。

# 10. 部署方案

## 10.1 推理流程

每个决策周期：

```text
1. AprilTag / 引擎得到车位姿
2. YOLO 更新 box/target ID
3. belief manager 做 ID 排除
4. 经典算法计算 BFS / legal push / dead corner / FOV candidates
5. 拼接 X 和 g
6. LAMP-Net 前向一次
7. legal mask + safety filter
8. 选择 macro-action
9. BFS 生成低层动作
10. 电机闭环执行
```

注意：**神经网络每个 macro-step 调一次，不必每个 PID tick 调一次。**

## 10.2 推理时延估计

在 OpenART mini 上，模型约 273 K int8 参数，主要算子是 depthwise separable conv 和 FC。考虑 MicroPython + TFLite Micro overhead：

| 部分                    |    mini 预计 |
| --------------------- | ---------: |
| 经典特征 BFS / legal mask | 0.5–1.5 ms |
| LAMP-Net INT8 推理      |    8–25 ms |
| action decode         |      <1 ms |
| BFS 低层路径              |     1–3 ms |
| 总决策开销                 | 约 12–30 ms |

保守估计仍小于 50 ms。

在 OpenART Plus 上，M7 1 GHz + M4 分担控制，预计推理更稳，可能在 6–15 ms 区间。

## 10.3 与 YOLO / CV 共存

RFC 提到同一 MCU 还要串行跑传统视觉、YOLO、决策模型，因此我建议：

1. **YOLO 不要每帧都跑**
   ID 一旦识别永久保存，只在未知实体进入 FOV 或识别置信度低时运行 YOLO。

2. **决策网络按 macro-step 运行**
   如果正在执行 BFS 路径，不需要每帧重新决策，除非定位偏差或碰撞失败。

3. **单 TFLite arena 复用**
   由于历史经验显示多模型并发不稳定，最好把 YOLO 和 LAMP-Net 串行执行，并明确释放/复用 tensor arena。RFC 中也提到应倾向单模型或必要时手写 INT8 推理。

4. **LAMP-Net 可手写 INT8 fallback**
   结构非常简单，depthwise conv + pointwise conv + FC 都容易手写或用 CMSIS-NN API 实现。

# 11. INT8 量化策略

训练时直接做 QAT，避免 post-training quantization 后策略崩掉。

建议：

1. 所有 Conv / FC 使用 fake quant。
2. 不使用 BatchNorm、LayerNorm、Softmax 依赖逻辑。
3. 输出 logits 保持 int8/ int16 accumulator，最后在 C 端只做 argmax，不必真的 softmax。
4. value head 可以量化到 int8 后映射回固定范围：

[
V \in [-128,127] \mapsto [-C_{\max}, 0]
]

预期量化损失：

| 方式                | 预期 win-rate 损失 |
| ----------------- | -------------: |
| PTQ               |      2–6 pp，不稳 |
| QAT               |       0.5–2 pp |
| QAT + action mask |     通常 <1.5 pp |

action mask 很重要，因为即使 logits 有小误差，非法动作也不会被选中。

# 12. 归纳偏置论证

## 12.1 信息瓶颈

原始状态中有大量无关信息。LAMP-Net 输入的是：

[
\text{state} = \text{grid geometry} + \text{belief} + \text{classical planning features}
]

它丢弃了图像纹理、AprilTag 细节、外围固定墙等无关信息，只保留决策变量。
这降低了样本复杂度，也降低了模型容量需求。

## 12.2 对称性

推箱子决策对平移、镜像、ID 重命名有天然对称性。空间化 action head：

[
Z_{\text{push}} \in \mathbb{R}^{H \times W \times 4}
]

比固定动作类别更保留平移等变性。ID permutation 增强则让模型学习“匹配关系”，而不是数字本身。

## 12.3 可微优化景观

纯 BC 的 one-hot 标签在多个近似最优动作存在时会制造冲突梯度。soft target：

[
q(a) \propto \exp(-C^*(s,a)/\tau)
]

可以让多个好动作共享概率质量，优化更平滑。

## 12.4 Generalization bound 直觉

泛化误差通常随假设空间复杂度增加而变差。LAMP-Net 通过三种方式缩小假设空间：

1. 用规则引擎排除非法动作。
2. 用 BFS/死角/距离场注入可计算结构。
3. 用小模型 + action mask 只学习动作排序。

因此它比“让 CNN+MLP 直接从状态猜动作”更容易泛化到新地图。

## 12.5 部分可观测性

POMDP 的核心是 belief state。LAMP-Net 明确维护：

[
B, T, M
]

并用 entropy / information gain 指导探索。这样它不会把“未知 ID”误认为一个普通类别。

## 12.6 配对组合爆炸

箱子—目标配对本来可能有 (N!) 种。LAMP-Net 用 soft matching matrix：

[
M_{ij} = \sum_k B_{ik}T_{jk}
]

把组合问题压成 (N^2) pairwise feature。最大 N=5 时非常小。

# 13. 风险点与备选方案

## 风险 1：Phase 6 炸弹决策仍不稳定

原因：炸弹有强长程效应，一次爆破会改变地图拓扑。

备选：

* 增加 bomb-specific value head。
* 对炸弹动作做 depth-2 symbolic rollout。
* 训练时过采样所有需要爆破的关卡。
* 对每个 bomb action 额外计算爆破后连通性增益：

[
G_{\text{bomb}} =
|\text{reachable after}| - |\text{reachable before}|
]

作为手工 bonus。

## 风险 2：探索动作浪费时间

原因：模型可能过度保守，先扫完所有 ID 再推箱。

备选：

* 只在以下情况强制探索：

  1. 当前计划需要确定 ID；
  2. belief entropy 高；
  3. 未知箱子即将被推向某目标；
  4. 排除法无法推出。
* 若某箱子与任何可能目标的最优动作一致，则允许先推。

## 风险 3：BC 学到专家偏差

BestFirst 不是全局最优，AutoPlayer 更偏启发式。

备选：

* 用 IDA* 只标注 hard states 和 validation failures。
* 用 ranking loss，而不是完全模仿某一条轨迹。
* teacher ensemble 生成 soft labels。

## 风险 4：TFLite Micro 实测比预期慢

备选模型 Tiny-LAMP：

```text
Input 10×14×48
1×1 Conv 48→32
DSConv blocks ×4
Global average pooling
FC 96→96→96
push/view heads
```

参数约 90–130 K，INT8 后约 120–180 KB。若主模型推理超过 50 ms，直接切 Tiny-LAMP。

## 风险 5：真实车执行误差导致状态偏离

备选：

* 每个 macro-action 执行后重新读取 AprilTag 状态。
* 若实际状态与预测状态不一致，立即重规划。
* 不让网络输出连续电机控制，只输出 macro-action，底层由闭环控制负责。


# 15. 一句话总结

这道题最适合的不是“大一统神经网络”，而是**小神经网络 + 强符号结构 + 合法动作 mask + belief state + DAgger**。
LAMP-Net 的核心优势是：让网络只学专家最难显式编码的“长期优先级”，而把 BFS、合法性、死角、ID 排除、FOV 信息增益这些确定逻辑交给经典算法。这样既符合 OpenART mini 的部署约束，也比普通 hybrid CNN+MLP 更可能在 Phase 4–6 上突破。
