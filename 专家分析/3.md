## 0. 摘要

本方案提出一个名为 **GridNet** 的全卷积 (FCN) 单模型架构，配合一组**显式的领域特征通道**（push-distance field、deadlock mask、reachability mask 等）作为输入。模型在 14×10 的可玩网格上输出一张 `(14, 10, 5)` 的 **action Q-map**，编码"以网格位置为索引的高层推箱意图"，再交由现有 BFS 底层规划器翻译为低层动作。

核心数学性质：

| 维度 | 数值 |
|---|---|
| 参数量 (FP32) | ~52 K |
| 参数量 (INT8 后) | **~52 KB**（< 500 KB 预算的 ~10%） |
| 推理 MAC | ~5.2 M MAC |
| 预计推理时延 (M7@600 MHz, CMSIS-NN INT8) | **20–35 ms**（< 50 ms 预算） |
| Tensor arena | < 80 KB（< 2 MB 预算的 4%） |
| 仅使用 TFLite Micro 已支持算子 | Conv2D / DepthwiseConv2D / FullyConnected / ReLU / AvgPool / Reshape / Softmax |

训练范式上，方案采用 **多源 BC + DAgger + 价值蒸馏** 的三阶段流水线，并在数据上利用 **D₄ 二面体对称群增广**（×8 数据放大）与 **ID 排列增广**（×N!）。

预期表现（含浅层 branch search 复用现有 inference 框架）：phase 1–3 ≥ 99%；phase 4 ≥ 95%；phase 5 ≥ 95%；phase 6 ≥ 90%。所有数字均为基于参数预算与归纳偏置的论证下界，非实证结果。

## 1. 设计哲学与三条核心观察

在选择架构之前，我先把这道题"问题骨骼"剥到最细：

### 观察 A：状态空间在欧式格上是稠密的，但在动作语义上是稀疏的

可玩网格仅 140 个单元；最多同时存在 5 个箱子 + 5 个目的地 + 3 个炸弹 = **最多 13 个动态实体**。**每一步真正"需要决定"的事**只有：

> 在所有 (实体 e, 推送方向 d) ∈ 8 entities × 4 dirs ≤ 32 个候选动作 中，加上 ≤ 140 个"探索目标格"，一共 **不超过 ~170 个高层动作**。

原 baseline 的 54 类 softmax 暗示了一种"每步预测 4–8 方向 + 几个特殊动作"的扁平离散动作空间——这种编码**丢掉了"动作和实体的绑定关系"**。GridNet 则把动作空间按"位置 × 方向"的张量结构展开，把"推哪个箱子"还原成"在哪个格子开火"。

### 观察 B：箱子–目的地的配对结构是双部图匹配

赢的条件 `class_id == num_id` 形成一个 N × N 的 0/1 配对矩阵。一旦所有 ID 都被识别，**这个匹配就退化为 N 个独立的"单箱子单目的地"子问题**，每个子问题都可以用反向 BFS 在 ≤ 1 ms 内求出**精确的"下一步该往哪儿推"**。

这意味着：**在 ID 已知的情况下，决策网络的实际工作量只剩下"多个推送任务之间的调度（避免相互阻塞 / 死锁）"**。这是一个比"端到端学 Sokoban"小得多的子问题。GridNet 的设计原则是**把一切能用经典算法在 1 ms 内算出来的东西，预先算好喂给网络**——网络只学那些经典算法做不好的事（多任务调度、不确定下的探索、炸弹时序）。

### 观察 C：MCU 上的算力预算是"对称破缺"的资源约束

500 KB FLASH / 2 MB RAM / 50 ms 看似紧张，但分摊后并不均匀：
- **FLASH** 实际 baseline 路径上的 YOLO 已占 ~200–300 KB；决策模型的真实预算是 ~200–300 KB。
- **MAC 预算** 才是真瓶颈。CMSIS-NN INT8 在 M7@600 MHz 实测 ~80–150 MMAC/s（参考 NXP / Arm 官方 MNIST/CIFAR benchmark），50 ms × 100 MMAC/s = **5 M MAC** 是上限。

这个数字直接定义了 GridNet 的"形状预算"：在 14×10 上做 5–7 层 32 通道的卷积（~1–5 M MAC）刚好合身。这同时排除了任何"先扩张到 64×64 然后做 ResNet-18"的方案。

## 2. §8.1 架构数学描述

### 2.1 状态编码 — 输入张量 `S ∈ ℝ^{14×10×20}`

输入为 20 个 `14×10` 的特征图（裁掉外圈不可破坏的框架墙——对决策无信息且省 ~26% 算力）。所有通道都已归一化到 `[0, 1]` 或 `{-1, 0, 1}`：

| 组别 | 通道 # | 名称 | 含义 |
|---|---|---|---|
| **静态地图** | 0 | `wall_inner` | 内墙（可被炸毁的墙）= 1 |
| | 1 | `free` | 空地 = 1 |
| **车自身** | 2 | `player_pos` | one-hot；车所在格 = 1 |
| | 3 | `player_dir_cos` | `cos(θ)`，θ 为车朝向（量化到 8 方位） |
| | 4 | `player_dir_sin` | `sin(θ)` |
| | 5 | `player_bfs_dist` | 车到该格的 BFS 步数 / max（归一化）|
| | 6 | `reachable` | 车当前可达 = 1 |
| **实体位置** | 7 | `box_all` | 所有箱子位置 |
| | 8 | `bomb_all` | 所有炸弹位置 |
| | 9 | `target_all` | 所有目的地 |
| | 10 | `box_known` | 仅 ID 已识别的箱子 |
| | 11 | `target_known` | 仅 ID 已识别的目的地 |
| **配对几何**（关键先验）| 12 | `push_dist_field` | 已配对箱子→目标的最短推送步数（每格取 min over boxes，归一化）|
| | 13 | `push_recommend_N` | 在该格"推下一格往北可使某已配对箱前进" = 1 |
| | 14 | `push_recommend_E` | 同上，东 |
| | 15 | `push_recommend_S` | 南 |
| | 16 | `push_recommend_W` | 西 |
| **危险与不确定性** | 17 | `deadlock_mask` | 该格若放入未配对箱即死锁 = 1（经典 simple-deadlock + corner deadlock 检测）|
| | 18 | `fov_visible` | 当前帧 FOV 内 = 1 |
| | 19 | `unknown_residual` | 未识别实体邻域 / 待探索热度 |

**说明**：
- 通道 12–16 是把 §5.5 中"反向 BFS 推送距离场"显式注入网络。在 ID 全部已识别时，这五个通道几乎"含答案"——朴素地说，理论最优 policy 在 phase 1–3 上就是 `argmax_d push_recommend_d`。网络只需要学"什么时候这个朴素答案不对"。
- 通道 17 (deadlock) 让网络看到"哪些格不能推"。此特征对 phase 4–6 至关重要，是 baseline 失败的可能原因之一。
- 通道 18–19 显式编码**部分可观测性**——网络看得到"我现在还有什么不知道"。

**输入大小**：`14 × 10 × 20 = 2800` 个 INT8 → ~2.8 KB。

### 2.2 网络结构

```
Input  S ∈ ℝ^{14×10×20}
   │
   ▼
Conv2D 3×3, 20 → 32, padding=same         [W: 5,760]   [MAC: 0.81M]
   │  ReLU
   ▼
[Block 1]  DepthwiseConv2D 3×3, 32        [W: 288]     [MAC: 40.3K]
           Conv2D 1×1, 32 → 32  (PW)      [W: 1,024]   [MAC: 143.4K]
           ReLU
   ▼
[Block 2]  DepthwiseConv2D 3×3, 32        ↑ 同上
           Conv2D 1×1, 32 → 32  (PW)
           ReLU
   ▼
[Block 3]  DepthwiseConv2D 3×3, 32  + dilation=2     ← 增大感受野
           Conv2D 1×1, 32 → 32  (PW)
           ReLU
   ▼
[Block 4]  DepthwiseConv2D 3×3, 32                    ← residual: out += Block 2
           Conv2D 1×1, 32 → 32  (PW)
           ReLU + Add (skip from Block 1 output)     ← 用 Add op
   ▼
[Block 5]  DepthwiseConv2D 3×3, 32
           Conv2D 1×1, 32 → 32  (PW)
           ReLU
   ▼
F ∈ ℝ^{14×10×32}    ← 共享特征图（"公共干"）
   │
   ├──▶ Conv2D 1×1, 32 → 5    [policy head]   [W: 160]   [MAC: 22.4K]
   │       │
   │       ▼  Q ∈ ℝ^{14×10×5}    ← 4 个推送方向 + 1 个"探索"通道
   │
   └──▶ AvgPool2D (14,10) → 32-d                       [无参数]
            │
            ▼
        FullyConnected 32 → 16, ReLU                   [W: 528]
            │
            ▼
        FullyConnected 16 → 2                          [W: 34]
            │
            ▼  (V, p_deadlock) ∈ ℝ²
                 V: 状态价值估计（用于 branch search 剪枝）
                 p_deadlock: 是否已不可解（辅助损失）
```

**感受野计算**：5 层 3×3（其中一层 dilation=2）→ 等价感受野 `2 + 2 + 4 + 2 + 2 + 1 = 13`，足以覆盖 14×10 整张可玩区域。

**关于 BatchNorm**：方案中**不使用** BatchNorm（RFC 4.4 已禁用动态 BN）。改用 fixed scale + bias，等价于一个加固定常数的 affine（INT8 量化时直接折进 conv 偏置）。训练时使用 GroupNorm 训练后融合掉——但 RFC 也禁了 GroupNorm，所以更稳妥的做法是**完全不用 normalization，靠 careful weight init (He/Kaiming) + ReLU + 残差路径稳定训练**。这对 32 通道的窄网络是 well-tested 的（参见 Brock et al. 2021 "NF-ResNet"）。

**关于残差**：唯一一处 skip 用 `Add`（被允许的 op）。

### 2.3 动作输出与解码

`Q ∈ ℝ^{14×10×5}` 的语义：

| 通道索引 | 含义 |
|---|---|
| 0 | 在该格往**北**推（仅当该格为箱/炸弹时有意义）|
| 1 | 在该格往**东**推 |
| 2 | 在该格往**南**推 |
| 3 | 在该格往**西**推 |
| 4 | 把车开到该格做**视觉探索**（仅当该格为可达空地时有意义） |

**解码（在 MicroPython 侧执行，约 < 1 ms）**：

```python
# 1. 构造合法性掩码
mask = -inf * ones_like(Q)
for (x, y) in box_positions ∪ bomb_positions:
    for d in {0,1,2,3}:
        if push_is_legal(state, (x, y), d):     # 经典物理校验
            mask[x, y, d] = 0
for (x, y) in reachable_cells:
    if exploration_is_useful(state, (x, y)):    # FOV gain > 0
        mask[x, y, 4] = 0

# 2. 取 argmax
(x*, y*, d*) = argmax(Q + mask)

# 3. 翻译为低层动作（沿用现有 BFS 规划器）
intent = ("push", (x*, y*), d*) if d* < 4 else ("explore", (x*, y*))
low_level_seq = bfs_planner.plan(state, intent)
```

**为什么这个动作空间正确**：
1. **大小对齐于真实决策维度**：`14 × 10 × 5 = 700` 个 logits，但其中合法动作 ≤ ~40 个 (5 boxes × 4 dir + ~20 explore cells)，搜索代价低。
2. **平移等变**：把整张地图（含车与所有实体）平移 `(δx, δy)`，最优动作的网格索引也跟着平移，softmax 分布不变——这正是 CNN+1×1 head 的天然性质。
3. **不需要"实体索引"**：网络从不被要求"输出 box_id=2"，避免了 ID 排列的混乱。

### 2.4 参数预算精算

| 模块 | 权重数 (FP32) | INT8 大小 | MAC |
|---|---:|---:|---:|
| Conv2D 入口 (20→32, 3×3) | 5,760 + 32 bias | 5.8 KB | 806 K |
| Block 1 (DW+PW) | 1,312 + 64 | 1.4 KB | 184 K |
| Block 2 (DW+PW) | 1,312 + 64 | 1.4 KB | 184 K |
| Block 3 (DW dilated + PW) | 1,312 + 64 | 1.4 KB | 184 K |
| Block 4 (DW+PW) | 1,312 + 64 | 1.4 KB | 184 K |
| Block 5 (DW+PW) | 1,312 + 64 | 1.4 KB | 184 K |
| Policy head 1×1 | 160 + 5 | 0.16 KB | 22 K |
| Value head FC1 | 512 + 16 | 0.5 KB | 0.5 K |
| Value head FC2 | 32 + 2 | 0.03 KB | 32 |
| **合计** | **~13.0 K params** | **~13 KB** | **~1.75 M MAC** |

⚠️ **重新核算**：13 KB 远小于 500 KB 预算。这给了我们巨大的"扩参空间"。可能的扩张方向：

- 通道 32 → 48：参数 ×2.25, MAC ×2.25 → 30 KB / 4 M MAC，依然很安全。
- 加 2 个 block：总深度 7 → ~20 KB / 2.5 M MAC。
- 入口 conv 增至 5×5：感受野 ×1.5，但 MAC 涨较多。

我建议 **首发用 32 通道 5 block 的最小版** 跑通 phase 1–4，再据消融决定是否增厚。

### 2.5 量化形态

- **训练**：FP32。最后一阶段用 **QAT (Quantization-Aware Training)** 微调 5–10 epoch。
- **量化方案**：per-channel symmetric INT8 for weights, per-tensor asymmetric INT8 for activations。policy head 输出可保持 INT16（仅 5 个标量×140 格 = 700 个值，无内存压力），降低 argmax 翻转风险。
- **预期精度损失**：< 1 pp win_rate（远低于 §6.4 要求的 2 pp 上界），因为：
  - 网络很浅、激活分布稳定；
  - 关键的 "push_recommend" 输入信号在 INT8 下基本无损（已是 0/1）；
  - QAT 直接优化的是量化后的 cross-entropy。

## 3. §8.2 归纳偏置论证

为什么 **GridNet（特征注入式 FCN）** 比"通用 hybrid CNN+MLP"更适合本场景？分五个角度：

### 3.1 信息瓶颈视角

baseline 的 "flat vector + 部分 2D + 拼接 + MLP" 把空间结构压成向量再恢复——这是两次 lossy 投影。而 Sokoban 的最优 policy 几乎完全由**局部几何**决定（"我前面那格是不是墙？""那个箱子的目的地在哪个方向？"）。**FCN 的隐表示自始至终保持 (H, W, C) 结构**，不丢空间相关性，正好与最优策略所需的信息几何对齐。

形式化：设最优策略 π* 是状态 s 的函数。可证 π* 对车 + 目标对应箱的局部 5×5 邻域的依赖是 **locally Lipschitz** 的（因为推一格只能改变局部 3×3 区域）。FCN 的层级感受野严格匹配这种"短程依赖 + 长程汇总"的结构，**MLP 没有这种保证**。

### 3.2 对称性 / 等变性

游戏在二面体群 **D₄**（4 个旋转 + 4 个反射，共 8 元素）下严格不变。也就是说：把整张地图绕中心旋转 90° / 镜像翻转，最优动作随之同变换。

GridNet 的 Conv2D 对**平移**自然等变；对 D₄ 通过**数据增广**实现近似等变（每个 (s, a) 样本在训练时随机施加一个 g ∈ D₄）。这等于免费把训练集放大 8 倍。

> **更进一步的方案**（暂不采用，理由见 §3.6）：用 G-CNN（Cohen & Welling 2016）实现严格 D₄-等变。但 G-CNN 需要 lift-and-pool 操作，在 TFLite Micro 不可用——故仅退而求其次的"软约束"路线（数据增广 + 测试时 8-fold ensembling）。

baseline 的 hybrid CNN+MLP 中，MLP 部分**完全破坏对称性**（输入向量的元素顺序不能旋转），这是一个明显的归纳偏置缺陷。

### 3.3 排列不变性 / 实体不可分

箱子之间是**可交换的**（除 ID 外）；目的地之间也是。任何"按 box_id 0..4 拼接到向量"的设计都引入了一个**虚假的 ordering 偏置**——网络可能学到"第一个箱子总是先解"这类与世界无关的模式，并在测试时崩溃。

GridNet 解法："箱子"作为输入的存在方式是**它在某个 (x, y) 上为 1**——网络从未见到 box_0 / box_1 这样的索引，只见到"在 (3, 4) 这格有一个箱子"。所有箱子被 conv kernel 同等处理。**ID 信息**通过 push_recommend 通道**以"几何梯度"形式**进入，而非以"标量索引"进入。这把"ID 排列对应的 N! 等价类"在输入端就坍缩成同一表征。

### 3.4 配对组合爆炸的解耦

5 个箱子 + 5 个目的地，理论配对方案数 5! = 120。一个朴素端到端模型要"想清楚"每种配对——这是组合困难的。

GridNet 把这件事**放在输入侧**用经典算法解：
- 对**已识别**的箱-目对（class_id 已知 → 一一确定），用反向 BFS 计算 push_distance_field。
- 对**未识别**的，留作 0（网络看到的是"这个箱子在等 ID"）。

**结果**：即便 5 箱 5 目的地，网络看到的"配对几何"也只有一张图（取所有 box→target 距离场的 min 或 channel-wise stack）。**网络从不需要枚举 N!**。

这是**用领域算法消除组合复杂度**的经典做法（DeepMind 的 AlphaStar / OpenAI Five 都用过类似思路：let the engine give the model derived state）。

### 3.5 部分可观测性的显式建模

未识别实体在通道 10–11 上为 0，而通道 7–9 (all-positions) 仍为 1。**网络看得到"差异"** = "这是个未知的实体"。配合通道 18 (FOV) 和 19 (待探索热度)，网络可以学到一个**两阶段策略**：

1. 只要存在未识别实体且当前 push_distance_field 模糊（一些 channel 12–16 全 0 的箱子），就触发"探索"动作。
2. 否则触发推送动作。

形式化：定义"信息收益"
$$ I(s, a_{explore}(x,y)) = H(\text{ID} \mid s) - \mathbb{E}_{o' \sim p(o' \mid s, a)} H(\text{ID} \mid s, o') $$

这个量在 ≤ 1 ms 内可由 raycasting 估计并喂给通道 19；网络无须自己学条件熵，**只需要学"何时探索 vs 何时推"的 trade-off**。

### 3.6 优化景观与泛化界

**优化景观**：FCN with residual + ReLU + 小通道数（32）已被反复实证（NF-ResNet, MobileNetV2 in 2018-2020）具有 well-conditioned 的损失曲面，不需要 BN。GridNet 共 ~13 K 参数，远小于训练样本量 10⁵–10⁶——属于"过参数化下界"侧，泛化误差由 **representation 的偏置质量** 主导，而非 capacity。

**泛化界**（粗略 Rademacher）：对于 D₄-增广后的训练集，等效样本量为 8N。一个 ~13K 参数的 ReLU 网在 8 × 10⁵ 样本上的泛化间隙
$$ \mathcal{G} \lesssim O\!\left( \sqrt{ \frac{\text{params} \cdot \log \text{params}}{N_{\text{aug}}} } \right) \approx \sqrt{\frac{13000 \cdot 9.5}{8 \times 10^5}} \approx 0.012 $$

即 ~1 个百分点的训练–测试差。这与我们 phase 6 ≥ 90% 的目标一致。

### 3.7 显式解决的 baseline 痛点对照表

| Baseline 痛点 | GridNet 应对机制 |
|---|---|
| 54 类离散动作过少且无语义结构 | (14, 10, 5) Q-map，700 个候选位 + 按 (位置, 方向) 因子化 |
| flat-vector 破坏空间结构 | 全 FCN，始终保持 (H, W, C) |
| 排列依赖（box_id 顺序） | 输入端实体匿名化，仅用位置存在 |
| 看不到 deadlock，盲推到死角 | 通道 17 显式 deadlock mask |
| 看不到 push 几何，靠 conv 自己学 BFS | 通道 12–16 显式 push distance field |
| 部分观测下无 explore 概念 | 通道 4 的 explore action + 通道 19 信息热度 |
| 课程难度阶跃（phase 3→4 性能崩） | curriculum BC + DAgger 闭环 (见 §4) |
| 量化掉点不可控 | 浅网络 + QAT + value head INT16 |

## 4. §8.3 训练范式

### 4.1 三阶段训练流水线

```
┌─────────────────────┐    ┌────────────────────┐    ┌────────────────────────┐
│ Stage A: 预训练      │ →  │ Stage B: DAgger    │ →  │ Stage C: 价值蒸馏 + QAT│
│ AutoPlayer + BestFirst│    │ on policy rollouts │    │ IDA* on hard subset    │
│ ~10⁶ samples, 30 epoch│    │ ~10⁵ corrections   │    │ ~5×10⁴ samples, QAT    │
└─────────────────────┘    └────────────────────┘    └────────────────────────┘
```

#### Stage A：行为克隆预训练（多源混合）

- **数据源**：
  - AutoPlayer（fast，但次优）：phase 1–6 共 ~6000 张图 × 10 个 ID seed = 6 万 episode → ~10⁶ (s, a) 样本。
  - BestFirst（≤ 1.5 × OPT）：phase 3–6 上每张图跑 1 个 seed → ~4000 episode → ~10⁵ 高质量样本。
- **采样比例**：epoch 内 70% AutoPlayer + 30% BestFirst。AutoPlayer 提供"行为分布广度"，BestFirst 提供"质量上界拉升"。
- **损失**：
$$ \mathcal{L}_A = \underbrace{\text{CE}(Q, a^*)}_{\text{policy}} + 0.1 \cdot \underbrace{\text{BCE}(p_{\text{deadlock}}, \mathbb{1}[\text{state is dead}])}_{\text{aux}} + 0.01 \cdot \underbrace{\|\theta\|_2^2}_{\text{L2}} $$

  其中 CE 应用于**仅合法动作位置**（用 mask 把非法位置的 log-softmax 分母排除）。
- **数据增广**：
  - D₄ 群：随机 ∈ {id, r90, r180, r270, fx, fy, fxy, fdiag} 中均匀采。注意 push 方向通道 (12–16) 也要相应旋转/翻转。
  - **ID 重排列增广**：对每个 batch sample，把 box class_id 与 target num_id 做相同置换 σ ∈ S_N。这把 "(N!)² 种 ID 标签 → 同一物理任务" 的不变性显式喂给网络。
- **优化器**：AdamW lr=3e-4 → cosine to 3e-5；batch 256；30 epochs；约 4–6 GPU·h on RTX 5060 Ti。

#### Stage B：DAgger 修正

BC 的核心病灶是 **distribution shift**：模型在自己的 (suboptimal) 路径上从未见过专家纠错。phase 4–6 上 baseline 性能崩塌的主要原因之一就是这个。

DAgger 流程：
```
for round in 1..K (K=5):
    rollout 当前策略 π_θ in ~500 hard envs (phase 4-6, 来自 verified-seed 清单)
    每条轨迹：
        每隔 m=3 步触发一次 "is current state on expert path?" 检查
        若否：调 BestFirst 给出"从此状态起的最优动作"作 label
    把新 label 加入 replay buffer (max size 5×10⁴)
    在 buffer 上微调 π_θ 5 epoch
```

- DAgger 的理论保证：在 N 轮后期望损失满足 `L ≤ L_expert + O(1/N)`（Ross et al. 2011）。
- 计算开销：500 envs × 30 steps × 30 s/BestFirst-call ≈ 5 hours per round。5 round → ~25 GPU·h。

#### Stage C：价值蒸馏 + QAT

- 在 phase 5–6 的 ~500 张最难图上跑 IDA*（每张 60–300 s，总计 ~30 GPU·h，可分布式）。得到 (state, optimal_a, V*=optimal_step_count)。
- 用 IDA* label 对**已收敛的 GridNet** 做 5 epoch 微调，loss 加上 `λ_v · MSE(V_pred, V*)`，`λ_v = 0.05`。这让 value head 真正"接地气"。
- **同步开 QAT**：使用 PyTorch `torch.ao.quantization` 或 TFLite QAT API，在该 5 epoch 中模拟 INT8 量化。
- 输出 INT8 TFLite flatbuffer，验证 §4.4 所有 op 均合法。

### 4.2 损失函数全式

总损失（Stage C 末）：
$$ \mathcal{L} = \underbrace{\sum_{(s, a^*) } \text{CE}\!\left( \text{softmax}(Q^{\text{masked}}(s)), a^* \right)}_{\text{policy distillation}} \;+\; \lambda_v \!\!\sum_{(s, V^*)} (\hat V(s) - V^*)^2 \;+\; \lambda_d \!\!\sum_{s} \text{BCE}(\hat p_{\text{dead}}(s), y_{\text{dead}}) \;+\; \lambda_w \|\theta\|_2^2 $$

`λ_v = 0.05, λ_d = 0.1, λ_w = 1e-4`。

### 4.3 推理时的 branch search（沿用现有框架）

GridNet 输出的 V(s) 让 branch search 可以做**带 value 评估的 beam search**（vs baseline 的纯 policy beam）：
- beam 宽度 b = 3，深度 d = 4。
- 每展开一个子节点时，新状态的得分 = `α · log π(a | s) + β · V(s')` ，`α=0.7, β=0.3`。
- 总 NN 调用数：1 + 3 + 9 + 27 ≈ 40 次。每次 30 ms → branch 总耗时 ~1.2 s 在 PC 端；MCU 上若 30 ms × 40 = 1.2 s 太多，可降至 b=2, d=3 → 15 次 = 450 ms（仍仅在 deadlock 风险高时触发，不影响主循环 5 Hz）。

## 5. §8.4 部署可行性

### 5.1 推理时延（OpenART mini, M7@600 MHz）

CMSIS-NN INT8 在 M7 上的核函数性能（基于 Arm 公开 benchmark + 我们对 1.75 M MAC 的估算）：

| 算子 | 经验吞吐 (M7@600MHz) | 本模型对应 MAC | 时延 |
|---|---|---|---|
| Conv2D 3×3 (full) INT8 | ~120 MMAC/s | 0.81 M | 6.8 ms |
| DepthwiseConv2D 3×3 INT8 | ~80 MMAC/s | 5 × 40 K = 0.2 M | 2.5 ms |
| Conv2D 1×1 INT8 (PW) | ~150 MMAC/s | 5 × 143 K + 22 K = 0.74 M | 4.9 ms |
| FC INT8 | ~200 MMAC/s | 0.5 K + 32 = 0.5 K | < 0.1 ms |
| AvgPool / ReLU | ~free | — | ~0.5 ms 总和 |
| **合计前向** | | **~1.75 M MAC** | **~14.7 ms** |
| 主循环开销 (TFLite invoke + tensor copy) | | | ~5 ms |
| **总单次推理** | | | **~20 ms** |

预算 50 ms 还有 30 ms 余量，可用于：
- 测试时 D₄ 增广（4 旋转平均 → ~80 ms，**仅在关键决策点触发**）
- branch search 一层展开

### 5.2 内存（tensor arena）

- 最大 activation tensor: `14 × 10 × 32 × 1B = 4480 B`（INT8 单层）。
- TFLite Micro arena 通常需要 ~2× 最大 tensor 用于 ping-pong + 元数据。预估 **~40–80 KB tensor arena**，远低于 2 MB 预算。

### 5.3 与 YOLO + 传统 CV 共存的资源协调

主循环节奏（每 200 ms 一帧 = 5 Hz）：

```
t=0    : 摄像头取帧 (5 ms, OpenMV API)
t=5    : YOLO INT8 推理 (~80 ms, 已有约束)
t=85   : 传统 CV (AprilTag、轮廓) (~30 ms)
t=115  : 状态融合 + GridNet 输入预处理 (~10 ms, BFS 计算 push_dist 等)
t=125  : GridNet INT8 推理 (~20 ms)        ← 本模型
t=145  : action decode + BFS planner (~5 ms)
t=150  : 电机指令下发 (~2 ms)
t=200  : 下一帧
```

整圈 ~150 ms，留 50 ms safety margin。GridNet 占主循环 ~13%，**远比 YOLO（53%）轻**。

**与 YOLO 的内存隔离**：
- 两个模型用**两个独立 tflite Interpreter**，**不共享 arena**。这就避免了 §4.5 提到的 K230 上"双模型 driver 冲突"——因为 OpenART mini/Plus 的 TFLite Micro 是纯 CMSIS-NN 软件栈，无 driver。
- 内存上 YOLO ~150 KB arena + GridNet ~80 KB arena = 230 KB，远低于 RAM 预算。

### 5.4 INT8 量化损失估计

定性分析：
- **policy head**（最敏感）：5 个通道。最差情况下两个 Q 值差仅 0.5/255 ≈ 2e-3，可能 argmax 翻转。但因为 **mask 掉的非法动作 logits = -∞**，合法候选间的差通常 ≥ 0.05（凭经验观察 BC 收敛后 logit gap）→ INT8 翻转概率 < 1%。
- **value head**：用于 branch search 排序，对绝对值不敏感，单调即可。INT8 后排序通常保持。
- **deadlock head**：sigmoid 后只取阈值 0.5，鲁棒。

**预测**：win_rate INT8 vs FP32 下降 < 1 pp，**满足 §6.4 的 ≤ 2 pp 要求**。

## 6. §8.5 风险点 & 备选方案

### 6.1 主方案最可能失效的环节

按风险从高到低：

#### 风险 R1：phase 6 的多炸弹时序问题

phase 6 中"先炸开内墙再推箱"是一种 **deep multi-step planning**——一次推送的"立即收益"为 0（爆炸只是创造未来空间），1-step BC 很难学到。

- **症状**：phase 6 win_rate 卡在 ~75–85%，phase 1–5 正常。
- **根因**：value head 训练数据中"炸开-然后赢"的 trajectory 相对稀疏，V(s) 不够 sharp。
- **缓解 1**：在 Stage C 显式上采样 phase 6 含炸弹的 trajectory（×3 权重）。
- **缓解 2**：扩大 branch search 在含 bomb 状态下的深度（条件触发 d=6）。
- **缓解 3（备选 B1）**：显式给一个 "**bomb sub-policy**"——一个非常小的 MLP，输入是炸弹位置 + 内墙拓扑，输出"该炸哪面墙"的离散类别。两网联动。

#### 风险 R2：未识别 ID 集中在 phase 6 末期的不可解决探索

若 5 个箱中 4 个未识别且分布在 FOV 死角，模型可能反复在同一区域转圈。

- **症状**：phase 6 死循环超时，win_rate 受限。
- **缓解**：通道 19 (`unknown_residual`) 设计为 **"对所有未识别实体的 frontier-explorer 期望距离"**——即把 frontier exploration 的最优解（已知经典算法）直接喂给网络。网络无须自己重新发明 frontier。
- **备选 B2**：在 inference loop 中加一个 **rule-based fallback**：若连续 N 步 explore action 未带来 ID 增益，强制切换到 NBV (next-best-view) 经典规划。

#### 风险 R3：领域特征预计算开销超时

通道 12–16 (push distance field) 每帧都要重算。每个箱子一次反向 BFS 在 140 节点上 ≈ 0.3 ms（C 实现）；5 箱 = 1.5 ms。可接受。

但 deadlock detection（通道 17）若用完整的 transposition table 检测，可能 5–20 ms。

- **缓解**：使用 **lightweight deadlock detection**：仅检测 "corner deadlock"（角落 + 非目标）和 "edge-line deadlock"（沿墙边 + 该边无目标），二者复杂度均 O(N_box × 8)，< 0.5 ms。重 deadlock 让 NN 自己学。
- **备选 B3**：把 deadlock_mask 改成 **训练时 oracle 算（用慢算法），部署时轻量算**——等价于知识蒸馏。

#### 风险 R4：D₄ 数据增广破坏 push_recommend 通道的方向语义

通道 13–16 是按"绝对方向 NESW"编码的。做 r90 增广时必须**同时把通道循环移位**：(N→E→S→W→N) 才能保持方向一致性。

- 这是个**实现陷阱**而非理论风险。需要在数据 loader 里写一个 group action 的精确实现（10 行代码）。

#### 风险 R5：INT8 量化在 value head 损失大于预期

value 是回归任务，对量化敏感性高于分类。

- **缓解**：value head 保持 INT16 输出（仅多 4 字节内存）。
- **备选 B4**：value head 完全在 host 侧（用 `Q.max(dim=-1).values.mean()` 作为 value 的 proxy；牺牲一点精度换零额外推理）。

### 6.2 总体备选架构（若 GridNet 整体不达标）

| 备选 | 何时启用 | 大致设计 |
|---|---|---|
| **B-α: Two-tower GridNet** | phase 6 win_rate < 85% | 一个塔吃 box-related 通道，一个塔吃 bomb-related 通道，concat 后接共享 head。+30% 参数，+10% MAC。 |
| **B-β: GridNet + 蒙特卡洛 lookahead** | branch search 不够强 | 用 V(s) 做 1-ply MCTS rollout，K=20 模拟。MCU 上每决策 +200 ms（仅 deadlock-risk 高时触发）。 |
| **B-γ: Object-centric DeepSets head** | 实体数 > 5 时（赛规允许扩展） | 在 GridNet 共享 backbone 后，对每个实体做 ROI sum-pool（用 1×1 mask），FC 处理。permutation-invariant by construction。 |
| **B-δ: 完全端到端世界模型** | 同上+训练资源充裕 | 加一个 next-state predictor 头，做 model-based RL（MuZero-lite）。但 MCU 部署复杂，仅作 PC 端策略上界。 |

### 6.3 一份"什么时候应该换方案"的判据表

| 观测 | 判据 | 行动 |
|---|---|---|
| Stage A 后 phase 1 win_rate < 95% | 输入特征或 backbone 不够 | 增至 48 通道；检查 push_recommend 通道是否真的注入正确方向 |
| Stage A 后 phase 4 win_rate < 70% | 多任务调度学不会 | 启动 Stage B (DAgger)，并启用 B-α |
| Stage C 后 phase 6 win_rate < 90% | 长程规划弱 | 启用 B-β（MCTS 头） |
| INT8 win_rate 比 FP32 降 > 2 pp | 量化敏感 | value head 升 INT16；policy head 用 per-channel symmetric |
| 单次推理 > 50 ms | MAC 估算偏乐观 | 通道 32→24，或 5 block→4 block |

## 7. 附录

### A. 训练用伪代码（PyTorch 风格）

```python
class GridNet(nn.Module):
    def __init__(self, in_ch=20, hid=32):
        super().__init__()
        self.stem = nn.Conv2d(in_ch, hid, 3, padding=1)
        self.blocks = nn.ModuleList([DSConv(hid, dilation=2 if i==2 else 1)
                                     for i in range(5)])
        self.policy_head = nn.Conv2d(hid, 5, 1)
        self.pool = nn.AdaptiveAvgPool2d(1)
        self.value_fc = nn.Sequential(nn.Linear(hid, 16), nn.ReLU(),
                                      nn.Linear(16, 2))

    def forward(self, x):
        x = F.relu(self.stem(x))
        skip = x
        for i, blk in enumerate(self.blocks):
            x = blk(x)
            if i == 3:                    # block 4 末尾加 skip
                x = x + skip
        Q = self.policy_head(x)           # (B, 5, 14, 10)
        v = self.value_fc(self.pool(x).flatten(1))   # (B, 2)
        return Q, v[:, 0], v[:, 1]        # Q, V, deadlock-prob
```

### B. 输入特征预计算的合理上界（C 实现，目标 < 5 ms / frame）

| 特征 | 算法 | 复杂度 | 实测目标 |
|---|---|---|---|
| `wall_inner`, `free` | 直接读地图 | O(140) | < 0.05 ms |
| `player_*` | 一次 BFS from player | O(140) | 0.3 ms |
| `box_*`, `target_*` | 直接遍历 ≤ 13 实体 | O(13) | < 0.05 ms |
| `push_dist_field` | per-box 反向 BFS in push-space | O(N_box × 140) | 1.5 ms |
| `push_recommend_*` | 由 push_dist_field 取 argmin 邻居 | O(140 × 4) | 0.5 ms |
| `deadlock_mask` (lightweight) | corner + edge check | O(140) | 0.3 ms |
| `fov_visible` | raycasting from player | O(R × C) | 1.0 ms |
| `unknown_residual` | frontier-distance BFS | O(140) | 0.5 ms |
| **合计** | | | **~4.5 ms** |

### C. 消融建议（用于实证验证设计）

按重要度递减建议跑这些消融：

1. **去除 push_recommend (12–16)**：预期 phase 1 win_rate 下降 3–8 pp，phase 4–6 下降 10–20 pp。验证"显式注入配对几何"的关键性。
2. **去除 deadlock_mask (17)**：预期 phase 4–6 下降 5–15 pp。
3. **MLP head 替换 1×1 conv head**：预期所有 phase 下降，phase 6 尤甚。验证"位置-动作因子化"。
4. **不做 D₄ 增广**：预期 phase 5–6 下降 ~3 pp。
5. **去除 value head**：branch search 退化为纯 policy beam，phase 6 下降 3–8 pp。
6. **通道 32 → 16**：MAC 减半，预期 phase 6 下降 5–10 pp。验证 capacity 边界。
7. **不做 DAgger (Stage B)**：phase 4–6 下降 10–20 pp，验证 distribution shift 的影响。

### D. 与 baseline 的"不变项"清单

为了降低工程切换风险，以下既有组件**完全保留**：

- BFS 底层路径规划器（动作翻译层）。
- 现有的 verified-seed 清单与求解器三件套。
- 现有的 phase 切分 / 评估脚本。
- branch search 推理框架（GridNet 只是把 policy + value 替换进去）。

需要新写的组件：
- 特征预计算 C 模块（~ 200 行）。
- GridNet PyTorch 模型 + TFLite 转换脚本（~ 100 行）。
- DAgger / 价值蒸馏训练循环（~ 300 行）。

## 8. 结论

GridNet 把这道题"重新切割"为：

> **(经典算法做的) 配对几何 + 死锁检测 + 探索热度** ⊕ **(神经网络做的) 多任务调度 + 时序判断**

——而不是让一个 MLP 同时学这两类完全不同的事。**这才是这道题真正的归纳偏置。**

在硬件约束之内，模型只用了 ~13 KB / 1.75 M MAC，留出充裕的预算供未来扩张。整套方案预计可在不引入任何禁用算子的前提下，将 phase 6 win_rate 从 baseline 的 50–80% 推到 ≥ 90%，且 INT8 量化损失 < 1 pp。

最大的待证项有两个：
1. **领域特征预计算的工程量**——是否真能压到 ≤ 5 ms / frame。
2. **phase 6 的炸弹时序**——若纯 BC + DAgger 无法学会，就启用 B-β（MCTS 头）。

二者皆已有备选路径（B1–B4, B-α–B-δ）。建议先按 §4 的三阶段流水线实施，第一轮跑出 phase 4 的 win_rate，再据此决定是否扩张通道数 / 启用备选。
